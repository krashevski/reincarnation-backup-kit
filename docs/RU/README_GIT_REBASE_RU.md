# 🧩 Шпаргалка: Как завершить git rebase при возникновении конфликтов

[![Лицензия: CC BY-SA 4.0](https://licensebuttons.net/l/by-sa/4.0/88x31.png)](https://creativecommons.org/licenses/by-sa/4.0/)

[🇬🇧 English](README_GIT_REBASE_EN.md) | [🇷🇺 Russian](../RU/README_GIT_REBASE_RU.md)

Этот документ объясняет, как безопасно разрешать конфликты слияния при git rebase. Он предназначен для новичков и используется в проекте Reincarnation Backup Kit.

## 1. 📌 Проверьте, какие файлы конфликтуют

Git автоматически выводит их список, но вы можете проверить вручную:
```bash
git status # проверьте, что рабочий каталог чист
```

Вы увидите файлы, помеченные как:
```bash
both changed: docs/RU/...
both changed: docs/EN/...
```

## 2. 📌 Откройте конфликтующий файл и удалите маркеры конфликтов

Откройте файл:
```bash
nano docs/EN/README_...
```

Ищите блоки типа:
```bash
<<<<<<< HEAD
Ваша версия текста
=======
Текст из другой ветки
>>>>>>> feature/update-readme
```

🎯 Выполните следующие действия:
Шаги:
1. Откройте файл в редакторе (например, nano):
```bash
nano docs/EN/README_GIT_BRANCHES_EN.md
```

2. Удалите маркеры <<<<<<<, =======, >>>>>>>.
3. Оставьте желаемый текст.
4. Сохраните и выйдите (Ctrl+O → Enter → Ctrl+X в nano).

## 3. 📌 Отметьте конфликт как решённый

```bash
git add docs/EN/README_GIT_BRANCHES_EN.md
git add docs/RU/README_GIT_BRANCHES_RU.md
```

## 4. 📌 Продолжите перебазирование

```bash
git rebase --continue
```

- Если есть ещё конфликты, повторите шаги 1–4 для них.

- Когда конфликтов больше не будет, перемещение будет завершено.

## 5. 📌 Полезные команды

❗ Отмена перемещения:
Если вы хотите отменить перемещение, используйте:
```bash
git rebase --abort
```

❗ Пропуск проблемного коммита:
```bash
git rebase --skip
```

## 6. 📌 Проверка статуса

```bash
git status
git log --oneline --graph --decorate -5
```

- Убедитесь, что ветка чистая и все коммиты применены.
- Все ваши коммиты README применены.

## 7. 📌 Последний шаг: отправьте обновлённую ветку на GitHub

```bash
git push --force-with-lease origin feature/update-readme
```

> --force-with-lease — если история переписана (из-за перебазирования), её можно использовать безопасно.

## Советы

- Всегда сначала используйте pull/fetch, чтобы минимизировать конфликты.
- Сохраняйте резервную копию локальных изменений (git stash), если вы в чём-то не уверены.
- Для текстовых файлов (README, документация) проще вручную объединить версии в случае конфликта.
- Используйте единый стиль коммитов и сообщений для удобства навигации.
